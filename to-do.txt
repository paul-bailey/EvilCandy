Awkward Things I'd Like to Improve
----------------------------------

* Every variable declared in a function has its own slot, regardless
  of scope.  This makes for a larger slot array than technically
  necessary, and keeps variables in memory for longer than they are
  needed.  Add scope tracking and shadowing,

* Built-in types like 'integer' and 'float' should appear to the user
  not as instantiation functions, but as classes.  I want a user to
  be able to do something like::

        class MyClass(float) {...

* Dot-naming duality for dictionaries is awkward.  I don't want to
  lose it, but I also don't like the inelegant work-around regarding
  namespace collision between a dictionary's items and its attributes,
  ie.: ``mydict['key']`` only retrieves an item whose ``key`` matches,
  while ``mydict.key`` first tries to retrieve attribute ``key`` and
  falls back on an entry if no such attribute exists.

* for-else loops are error-prone, since an if-for-else and for-if-else
  sequence without braces can be misleading; user will indent according
  to intention but not according to how the program will actually
  interpret it.  Consider a different keyword than 'else' in the for
  loop, maybe 'except'.
