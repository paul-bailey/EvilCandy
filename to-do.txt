Bugs
----

* BUGs 3/2026:

        * In interactive mode, a syntax error sometimes lets the
          remainder of the line be parsed from where the error occurred.

        * **fixed** In interactive mode, cannot assign a name when in a program flow
          statement.

        * **I'll tolerate it** (pseudo-bug) illusion of memory leaks when exiting
          via UAPI exit().  Not all of the breadcrumbs are accounted for.  See
          where suspected, vm.c do_call_func().

        * **fixed** segfault when f-string with conversion is the only part of a
          statement, eg.
                f'something {x}';
          but not if surrounded with parentheses:
                (f'something {x}');

* BUGs 2/2026:

        * **fixed** User cannot set a dictionary's property, because the var.c
          algo will think it's trying to set a dictionary field.

        * **fixed** When using readline, ^D is not interpreted as EOF during the
          first prompt.

        * **worked around** assert fails in var_sanity_check_types_tbl() when exiting with
          the exit() function instead of normal end of script.

* BUGs 6/2025:

        * "global x = ...something which causes error" is unpredictable
          whether x is declared or not after the exception is thrown.

* BUGs 5/2025:

        * Object mem leak, four not garbage-collected whenever there's a
          SyntaxError exception.  Other exceptions are fine.

* BUGs 4/2025:

        * **worked around** All user-visible data types need a .cpmz method.
